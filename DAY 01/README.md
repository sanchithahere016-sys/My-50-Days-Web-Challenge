# 📅 Day 01 – Semantic HTML Architecture

## 🚀 Challenge
Build the foundation of a community landing page using Semantic HTML without using unnecessary `<div>` elements.

## 🛠️ What I Built
- Created a semantic HTML webpage structure.
- Added a Header with Navigation.
- Built a Hero section with a heading and call-to-action.
- Created About, Initiatives, and Core Team sections.
- Added a Footer with copyright and social media links.

## 📚 What I Learned
- The importance of Semantic HTML in web development.
- How to use semantic tags such as `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, and `<footer>`.
- Why semantic elements improve accessibility and SEO.
- How to organize a webpage using a logical HTML structure before applying CSS or JavaScript.
- The difference between semantic elements and generic `<div>` containers.

## 💡 Key Takeaway
A well-structured HTML document is the foundation of every website. Building the structure first makes the webpage more accessible, maintainable, and easier to style in later stages.
